Architecture

모듈화 설계

단순 스크립트가 아니라 agents/domains 구조로 역할을 분리해 유지보수성을 높였습니다.

🎭 agents/

scene.py
일기 텍스트 → GPT 호출 → 장면 4개 JSON 추출. 구조화된 출력(Structured Output) 활용.
image.py
장면 JSON → gpt-image-1 API 호출 → 이미지 파일 저장. 배치 처리로 4장 순차 생성.

🎨 domains/

product_prompts.json
도메인별 프롬프트 설정 분리. shot(close-up), lighting(studio), mood(clean) 등 스타일 설정을 코드 밖으로 분리.
도메인 확장 가능 구조
product 외에 wedding, travel, food 등 새 도메인을 JSON 파일만 추가해 확장 가능.

🛡️ guardrails.py

check_max_iter()
무한 루프 방지. 최대 반복 횟수 초과 시 폴링 중단.
check_timeout()
시간 초과 방지. 영상 생성 대기 시간 제한.
check_predicate()
상태 조건 검사. 완료/실패 상태를 추상화해 조건 분리.

❌ Before — 첫 번째 시도

문제
캐릭터 외형 고정값 없이 장면 설명만 프롬프트에 입력했더니 4장 모두 다른 캐릭터가 등장했습니다.
결과
Scene 1은 남성, Scene 2는 외국인 어린이, Scene 3는 노인 등 — 같은 일기의 주인공인데 전혀 다른 인물이 등장해 스토리가 이어지지 않았습니다.
원인 파악
이미지 생성 모델은 각 장면을 독립적으로 생성하기 때문에 캐릭터 일관성을 유지하려면 프롬프트에 명시적으로 고정해야 함을 인식했습니다.

✅ After — 프롬프트 개선

해결 방법
모든 장면 프롬프트에 캐릭터 고정값을 추가했습니다.
"A young Asian woman with short black bob hair, wearing an orange sweater, watercolor style"
결과
4장 모두 단발머리 아시아 여성으로 일관된 캐릭터가 등장했습니다. 스토리가 자연스럽게 연결되는 그림일기가 완성됐습니다.
배운 점
이미지 생성 AI는 컨텍스트를 공유하지 않습니다. 일관성이 필요한 요소는 반드시 프롬프트에 명시적으로 고정해야 한다는 것을 직접 경험했습니다.
