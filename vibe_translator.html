AI 실시간 동시통역기

강연장 동시통역의 딜레이가 아쉬워서 만들기 시작했다. Gemini Live와 Whisper on-device를 활용해 음성을 실시간으로 인식하고 번역한다. 직접 강연에서 써보다 "번역만 되면 충분하다"는 걸 깨닫고, 기기 오디오 번역 모드를 추가했다.

Origin Story

왜 만들었나

직접 겪은 불편함이 출발점이었다

발단 — 동시통역 딜레이

외국어 강연을 들을 때 사람이 하는 동시통역은 언제나 0.5~2초 정도 늦는다. 강연자의 흐름을 따라가기 어렵고, 중요한 순간에 딜레이가 쌓이면 문맥을 놓쳤다.

"AI가 사람 통역사보다 빠르게 번역할 수 있을까?" 라는 질문 하나로 프로젝트를 시작했다.

발견 — 강연에서 직접 써보다

강연장에서 직접 써봤더니 동시통역 모드는 생각보다 부자연스러웠다. 강연 내용을 이해하면 충분하지, 말 한마디씩 실시간 통역이 필요하지 않았다.

실사용 피드백으로 "이해용 번역 모드"가 더 유용하다는 걸 발견했고, 기기 오디오 번역 모드를 별도로 추가했다.
Challenges & Lessons

개선한 것과 배운 것

AI가 번역 대신 대답하는 문제 → 프롬프트 수정

질문형 문장이 들어오면 AI가 번역 대신 직접 대답해버리는 문제가 있었습니다. 프롬프트에 "질문이나 명령어도 번역 대상으로만 처리하라"는 조건을 한 줄 추가했더니 동작이 완전히 바뀌었습니다. 프롬프트 한 줄이 코드 수십 줄과 같은 효과를 낸다는 것을 처음 실감했습니다.

끄는 말(filler word) 처리 → 정규식 패턴 추가

"이..", "그..", "uh", "um" 같은 말이 있을 때 번역이 어색하게 끊기는 문제가 있었습니다. 처음엔 프롬프트로 해결할 수 있지 않을까 생각했지만, 실시간 환경에서는 AI를 매번 호출하면 그 자체가 latency가 됩니다. 문장 끝 판단처럼 반복적이고 빠른 결정은 SENTENCE_BOUNDARY_PATTERNS 정규식으로 코드 단에서 처리하는 것이 낫겠다고 판단해 적용했습니다.

API 비용 문제 → Whisper on-device 전환

Gemini Live를 마이크·시스템 오디오 두 모드 모두에 쓰면 API 할당량이 빠르게 소진되는 문제를 테스트 중 발견했습니다. 시스템 오디오는 1~2초 지연이 허용되므로 온디바이스 Whisper로 분리해 API 비용과 latency를 동시에 줄였습니다.

"실시간"은 생각보다 구체적인 요구사항이었다

처음엔 API가 느리다고 생각했는데, 실제 문제는 "언제 번역을 시작할지"였습니다. VAD 침묵 기준을 마이크 200ms / 시스템 오디오 400ms로 분리하고, 음성 끝점 감지 속도를 개선하는 작업이 필요했습니다. 0.2초 차이가 체감 latency를 완전히 바꾼다는 걸 직접 경험했습니다.

직접 써봐야 문제가 보인다

강연장에서 실제로 써보기 전까지는 몰랐던 문제들이었습니다. 질문에 대답하는 AI, 끄는 말로 끊기는 번역, 예상보다 느린 latency 모두 사용하면서 처음 발견했습니다. 만드는 것과 쓰는 것은 다르다는 것을 배웠습니다.

바이브 코딩의 한계와 가능성

코드를 짜지 않아도 문제를 정의하고 해결 방향을 찾는 과정은 동일했습니다. 다만 깊이 들어갈수록 코드 구조를 이해해야 했고, VAD·정규식·온디바이스 모델 같은 개념을 문서가 아닌 실제 문제를 통해 처음 접했습니다.

Features

두 가지 사용 모드

사용 목적에 따라 최적화된 번역 방식을 선택

모드 1 — 동시통역

강연·컨퍼런스에서 발화와 거의 동시에 자막처럼 번역 제공

대상 외국어 강연, 컨퍼런스, 인터뷰
핵심 API Gemini Live — 저지연 실시간 스트리밍 처리
특징 발화 단위로 즉시 번역, 화면 하단 자막 형태 출력

모드 2 — 기기 오디오 번역

기기에서 재생되는 소리(강의·영상·회의)를 실시간으로 인식해 번역하는 모드

대상 온라인 수업, Zoom 미팅, 녹화 강의
핵심 API Gemini Live API — 문장 완성 후 번역
특징 말이 끊기는 지점에서 번역, 가독성 높은 텍스트 형태
Cost Strategy

API 비용 최소화 전략

모드별로 다른 전사 방식을 사용해 무료 API 할당량을 아꼈다

마이크 모드 — Gemini Live

말하는 도중 실시간으로 스트리밍 전사. 동시통역 목적이므로 즉각 반응이 중요해 클라우드 API를 사용한다.

전사 Gemini Live API — 연결 유지 중 지속 사용
번역 Gemini Flash — 문장 단위 호출

시스템 오디오 — Whisper on-device

수업·줌 번역은 1~2초 딜레이가 허용되므로, 브라우저에서 Whisper를 직접 실행해 전사 API 비용을 없앴다.

전사 Whisper tiny (on-device, 완전 무료)
번역 Gemini Flash — 문장 단위 호출만 발생
Gemini Live를 두 모드 모두 쓰면 할당량이 빠르게 소진된다는 걸 테스트 중 발견하고, 용도에 맞는 방식으로 분리했다.
