SNS 콘텐츠 자동 생성

플랫폼마다 콘텐츠를 따로 써야 하는 번거로움을 없앴다. URL 하나를 입력하면 Gemini API 3단 프롬프트 체이닝으로 LinkedIn·X·Instagram에 최적화된 콘텐츠를 동시에 생성한다. 각 플랫폼의 알고리즘 특성(북마크 가중치, 체류시간, 저장율)을 프롬프트 설계에 직접 반영했다.

한계와 배운 점

⚠️ 현재 모델의 한계

프롬프트 의존적 품질 불안정

같은 URL을 넣어도 viral_score나 포스트 톤이 매번 다르게 나올 수 있고, 이를 제어할 메커니즘이 없습니다.

단방향 콘텐츠 타입 지원

YouTube 자막과 블로그 본문만 처리 가능합니다. 자막이 없는 영상, 로그인이 필요한 페이지 등 실제 유저가 넣을 법한 URL의 상당수는 오류로 떨어집니다.

바이럴 스코어의 허구성

LLM이 스스로 생성한 포스트를 스스로 채점하는 구조입니다. 실제 인게이지먼트 데이터 없이 자기 평가하는 수치라 신뢰 근거가 없습니다.

💡 개선 방향과 배운 점

개선 방향

Few-shot 예시 포스트를 프롬프트에 삽입해 출력 품질 편차를 줄이는 방법, 실제 LinkedIn/X API로 게시 후 조회수를 역피드백하는 검증 루프 추가가 가능합니다.

그럼에도 의미 있는 이유

3단계 프롬프트 체이닝과 병렬 실행 구조를 직접 구현했고, 해커톤 당일 URL 입력부터 UI 출력까지 동작하는 End-to-End 파이프라인을 완성했습니다.

배운 것

프롬프트 설계가 곧 비즈니스 로직이라는 것을 체감했습니다. 로컬과 배포 환경은 다르다는 것을 직접 겪었고, IP 차단 우회 방법으로 RapidAPI 같은 중간 레이어를 활용하는 선택지가 있다는 것도 배웠습니다.
